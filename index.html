Why π and π² are not external mathematical constants

A frequent question: aren't π, 2/π, 1/π² simply imposed mathematical constants? In this framework they are derived rectification constants (§5.1, §5.2), not inputs:

  1. 2π emerges from rotational closure of phase variables — full-cycle angular states return to the same configuration. Lattice-symmetry property, not an input.
  2. Uniform measure 1/(2π) emerges from maximum entropy under "no a priori bias" (§5.2.5). Derived from an information-theoretic principle, not assumed.
  3. α = ⟨|cos θ|⟩ = 2/π from full-cycle absolute rectification (§5.1).
  4. δ = ⟨[cos θ]+·[cos φ]+⟩ = 1/π² from two independent half-wave constraints with product measure (§5.2).
  5. 4π in shell counting emerges from 3D isotropic averaging (§14.3); anisotropic regimes give different values (§17.1).

So π and its powers are outputs of rotational symmetry plus maximum-entropy uniform distribution, in the same operational sense that Kolmogorov scales emerge from turbulence dynamics rather than being imposed.

Gravity: lattice yield model (Appendix G)

Gravity is reframed as the restoring pressure of a jammed lattice attempting to fill the mass-equivalent void. The lattice has finite yield curvature demand Ψyield, so contact-mode acceleration cannot exceed a saturation limit:

2-channel decomposition (Appendix G.1):
  gobs = ggeom(R) = GM/R²    orbit / free-fall mode
  gobs = grestore ≤ g★    contact / static mode
Yield limit: g★ ≡ c² · Ψyield

Falsifiable predictions (distinguishable from Newton)

  • Solid-surface bodies with gpot > g★: scale-measured surface acceleration saturates at g★ rather than growing without bound.
  • Gas giants (Jupiter, Sun): no static solid-surface contact → no g★ measurable; only ggeom from orbital data. Surface-gravity comparisons across Earth-like and gaseous bodies are not apples-to-apples in this model.
  • Neutron-star regime: fluidization transition above g★; excess curvature demand dispersed into pressure / EM / rotation channels.
Honesty note on Ψyield Ψyield is currently a LOCK input calibrated to surface conditions, not a closed-form derivation from the VP unit. Earth's surface g ≈ 9.8 m/s² is therefore a consistency anchor, not a prediction. The predictions of the model are the structural features — 2-channel decomposition, saturation behavior, regime-dependent observables — which are independent of the specific Ψyield value. A closed-form derivation of Ψyield from VP geometry alone is an open item in §17.3 (vNext roadmap).

Methodology: LOCK → Derive → Gate

Every numerical statement in the whitepaper carries three traceable identifiers: (i) LOCK provenance, (ii) derivation chain, (iii) Gate verdict. Sentences without all three are not promoted to conclusions.

LOCK

Definitions, canonical inputs, thresholds frozen at a single source (SSOT). Changes exist only as new versions; old conclusions remain bound to the original LOCK.

Derive

Only LOCK items + permitted transformations. Re-deriving the same constant elsewhere is forbidden; the SSOT principle is enforced.

Gate

PASS / FAIL / INCONCLUSIVE verdicts only. Required Gate stack: G-SYM, G-LOCK, G-REG, G-RECT, G-STR, G-RCROSS, G-REP.

No-Tuning

Post-hoc adjustment of definitions, inputs, or thresholds is forbidden within a version. Selection bias requires pre-registered selection rules.

Verification FAQ

Isn't using cref = 299,792,458 m/s as an input circular?

Partly — for SI conversion, yes. The cref value is an SI-defined anchor (BIPM 1983), so SI values like "mp ≈ 0.938 GeV" are anchor-conditional unit conversions. However, the genuine predictions are dimensionless ratios: mp·rp/(hc) = 2/π, mp/me = 2π·νp,can, mH/Ulat = 1/(5π), αem−1 = 137.0364. In these, c cancels algebraically. The dimensionless agreement at 4–5 digits across multiple independent measured ratios is what the theory predicts; SI absolute values are unit conversions on top of that. This separation is made explicit in the table above.

Are there free fit parameters in the closed-form derivations?

For the dimensionless invariants in the table (αem, mp·rp/hc, mp/me, mH/Ulat, KC/hc): zero free parameters in the closed-form chain. External SI anchors (h, cref, λHeNe, rp) are SI/CODATA standards, not adjustable knobs. Internal LOCK inputs (Danch, A) are deterministic simulation outputs from lattice dynamics (analogous to Kolmogorov scale or jamming density φJ), not adjustable. Items with known limitations (me direct form, Ψyield, Ncore=82 origin) are explicitly flagged in §17.3.

How can αem−1 = 137.0364 emerge from integers alone?

Only integers and π enter the formula: 4π·(7+3+1 − δproj), with δproj = (35/32)·(2/π²)·(3/7). The 7 = 2+4+1 is the 3D minimal cancellation count (§8.2: pair + tetrahedral quad + survivor). The 3 is topological enclosure (§7.1). The 35/32 = (7!/3!2!2!) / (3·2⁶) is a permutation-over-projection ratio. The full chain is in §14.5 with deterministic verification scripts in the DOI bundle. Coincidence-fitting cannot achieve 0.0003% match across this many simultaneous constraints from a single backbone.

Why is gravity bounded near 9.8 m/s² on Earth-like bodies?

The model claim: contact-mode acceleration saturates at g★ ≡ c²·Ψyield; Earth happens to lie near saturation (Ψgeom ≈ Ψyield). The model is falsifiable on bodies where Newtonian extrapolation gives gpot ≫ g★ but no static surface contact exists (gas giants) or fluidization sets in (neutron stars). Note: Ψyield is currently a LOCK input, so 9.8 itself is a consistency anchor; the falsifiable content is the saturation behavior, not the specific number.

How does this differ from numerology or coincidence-fitting?

Three structural protections: (i) No-Tuning rule — definitions, inputs, thresholds cannot change after seeing results, only via versioning; (ii) SSOT — each constant is derived at exactly one location with no parallel re-derivations; (iii) multiple independent precision tests on the same backbone — αem (5-digit), mp·rp/hc (4-digit), mp/me (4-digit), mH (3-digit). Numerology fits one number at a time with adjustable functional form; here multiple independent measured ratios are simultaneously matched from a single backbone with closed forms.

How is "numerical agreement is not justification" enforced?

From §17.3 and §0.1.5: agreement with external numbers is recorded only as a Gate verdict item or error-budget report; it is never used to declare axioms or definitions "true." The reverse pattern — adjusting LOCK after seeing agreement — is forbidden by No-Tuning and triggers FAIL-NT (§1.1). This rule is a structural anti-fit safeguard.
